Описание датасета: Hepatitis C Virus (HCV) Data
Ссылка на датасет: UCI Machine Learning Repository: HCV Data

Общая информация
Этот датасет содержит данные лабораторных анализов крови пациентов, включая как здоровых людей, так и пациентов с гепатитом C разных стадий. 

Цель для кластеризации
Основная задача при использовании этого датасета для кластеризации — обнаружить скрытые группы (кластеры) пациентов на основе их биохимических показателей без использования заранее известных меток диагноза. Это позволяет проверить, могут ли алгоритмы машинного обучения:

Автоматически выделить группы, соответствующие разным стадиям заболевания.

Обнаружить аномалии (например, пациентов с нетипичными показателями).

Подтвердить или оспорить существующую медицинскую классификацию на основе объективных данных.

Описание признаков (атрибутов)
Датасет содержит 13 числовых атрибутов, которые являются идеальными входными данными для алгоритмов кластеризации.

Атрибут	Описание	Важность для кластеризации
Category (Целевой)	Диагноз пациента: 0=Донор крови, 0s=Подозреваемый, 1=Гепатит, 2=Фиброз, 3=Цирроз	Это целевая метка.
Age	Возраст пациента	Может влиять на формирование кластеров, так как некоторые заболевания более вероятны в определенных возрастных группах.
Sex	Пол пациента (м/ж)	Номинативный признак, требует преобразования (например, в 0/1).
ALB	Альбумин	Важный показатель функции печени.
ALP	Щелочная фосфатаза	Фермент, уровень которого может повышаться при заболеваниях печени.
ALT	Аланинаминотрансфераза	Ключевой фермент для оценки повреждения печени.
AST	Аспартатаминотрансфераза	Еще один важный фермент, указывающий на повреждение печени.
BIL	Билирубин	Продукт распада гемоглобина; высокий уровень вызывает желтуху.
CHE	Холинэстераза	Фермент, синтезируемый в печени; его уровень снижается при нарушении функции.
CHOL	Холестерин	Уровень может быть снижен при тяжелых заболеваниях печени.
CREA	Креатинин	Показатель функции почек.
GGT	Гамма-глутамилтрансфераза	Фермент, чувствительный к алкогольному и другому повреждению печени.
PROT	Общий белок	Отражает общий белковый обмен.

Ключевые особенности для анализа
Разнородность кластеров: Ожидается, что алгоритмы смогут четко отделить кластер здоровых доноров (Category=0) от кластеров пациентов с заболеваниями.

Градация заболеваний: Интересной задачей является проверка, смогут ли алгоритмы различить более тонкие градации, такие как Фиброз (2) и Цирроз (3), которые являются последовательными стадиями поражения печени.

Проблема "Подозреваемых": Группа "Подозреваемые" (0s) представляет собой интересный вызов для кластеризации.

Алгоритмы для применения

K-Means: Классический алгоритм для поиска сферических кластеров.

DBSCAN: Полезен для поиска кластеров произвольной формы и обнаружения выбросов. 

Agglomerative Clustering (Иерархическая кластеризация): Позволяет построить дендрограмму для визуального анализа иерархии кластеров.

MeanShift, Optics

Метрики оценки

Поскольку истинные метки известны, можно использовать внешние метрики для оценки качества кластеризации:

Adjusted Rand Index (ARI)

Adjusted Mutual Information (AMI)

V-score

Также использовались внутренние метрики:

Silhouette Score

Calinski-Harabasz Index

Davies-Bouldin Index
